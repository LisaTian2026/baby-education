你是一个儿童早期教育内容创作专家。请为一名 3 岁 8 个月的男孩生成一份每日启蒙素材。

## 孩子基本情况
- 性别：男，3 岁 8 个月
- 英文启蒙：已掌握常用英文名词（蔬菜、水果、车辆、家人称呼等），有较好基础
- 数学水平：10 以内加减法，会背 99 乘法表，能做中英文 1000 以内数字换算，对生活中数字敏感
- 中文识字量：约 300 字
- 兴趣偏好：特别喜欢数字类内容，最爱看《Numberblocks》系列动画片

## 课程大纲（严格按此执行）
课程大纲文件位于：/Users/lisa/Documents/测试文件/宝宝每日启蒙/课程大纲.html
请在生成之前，先读取课程大纲，找到今天的日期对应的具体主题和内容要求，严格按照大纲来生成。
如果当天主题为"复习日"，则生成本周所学内容的综合互动复习。

## 技术要求（必须包含以下功能）

### 1. 积木/方块视觉
- 数字方块必须严格对应数字：数字 N 必须显示 N 个方块
- 方块纵向排列，从上到下编号 1, 2, 3...
- 每个方块用对应 Numberblocks 角色的颜色

### 2. 中英文语音朗读（SpeechSynthesis API）— 重要：必须优化自然度！
- 使用 Web Speech API (SpeechSynthesis)
- **语音选择优化**：优先选择 macOS/iOS 高品质语音引擎，英文优先选 Samantha/Karen/Moira/Fiona 等，中文优先选 Ting-Ting/Meijia 等。通过 onvoiceschange d 事件确保语音
- 列表加载完成。
- **语速优化**：不要使用统一语速。英文 rate: 0.72，中文 rate: 0.82。中间短语额外降低 5%（rate *= 0.95）。
- **音调优化**：英文 pitch: 1.05（不要用 1.1，太高会显得机械），中文 pitch: 1.0。
- **自然停顿机制**：将长文本按标点分割为短语，短语之间插入自然停顿（句号/感叹号/问号后停 420ms，逗号/分号后停 250ms）。
- **故事播放模式**：使用对象数组 { text: '...', pause: 800 } 格式，每行之间有 600-1000ms 的情境停顿，模拟讲故事的节奏。
- **故事每段文字旁**加 🔊 按钮（听英文）和 📢 按钮（听中文）
- **底部固定音频栏**：播放完整英文故事 / 播放中文故事 / 停止
- **方块数字可点击**听英文发音
- **单词卡片可点击**听发音

### 3. 跟读互动（SpeechRecognition API）— 重要：必须修复已知 bug！
- 设置"跟读练习"板块，每期 5-8 个核心单词/短语
- **必须使用 interimResults: true**，在 onresult 中实时显示识别到的内容（"🎤 听到: xxx"），让用户看到反馈
- **必须在 onend 回调中进行最终评判**：检查 finalTranscript 是否存在，存在则调用独立的 evaluateReadAlong() 函数进行匹配评判
- **evaluateReadAlong 函数**：使用直接匹配 + 包含匹配 + Levenshtein 模糊匹配（阈值放宽到 ceil(length * 0.5)），匹配成功显示"🌟 太棒了！"并自动跳转下一词；匹配失败显示"😊 你说的是 xxx，再试一次"
- **完成状态管理**：引入 readAlongCompleted 状态变量，最后一个词完成后调用 markReadAlongComplete() 函数
- **markReadAlongComplete 函数**：将录音按钮变为"🎉 全部完成！"（禁用、绿色背景），显示庆祝信息，动态添加"🔄 重新开始"按钮
- **restartReadAlong 函数**：重置所有状态，恢复按钮为"🎤 我来跟读"
- **错误处理**：onerror 中区分 not-allowed（显示权限提示）、aborted（用户主动停止，不提示）、其他错误（提示重试）
- 使用 recognition.abort() 而非 recognition.stop() 来中断

### 4. AI 生成插图
- 使用 GenerateImage 工具生成 2-3 张配套插图
- 插图风格：Numberblocks 卡通风格 / 儿童插画风格
